# Contributing to Skedulo Developers Repositories

We welcome contribution to Skedulo Developers repositories, be that through feature suggestions, ideas, bug fixes or pull requests. We want the example code and other projects here to be as as helpful possible and you can help to ensure that.

## Code of Conduct

This project and everyone participating in it is governed by our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior to [developers@skedulo.com](mailto:developers@skedulo.com).

## How Can I Contribute?

There are many ways to contribute:

- **Report bugs** - Help us identify and fix issues, please use the GitHub issues tab within the relevant repository
- **Suggest enhancements** - Share ideas for new features, or other examples you'd like to see here
- **Submit code** - Fix bugs or implement features

## Reporting Bugs

**Before Submitting a Bug Report**

* Ensure you have the latest version of the repository (e.g run `git pull`)
* Check the documentation both on the repository, and at [developers.skedulo.com](https://developer.skedulo.com/)
* Search existing issues in the repository to avoid duplicates
* For issues with Skedulo or the Skedulo Pulse Platform, please raise a case with [support](https://support.skedulo.com/skedulo/directories)

**How to Submit a Bug Report**

Create an issue in the relevant repository and include:

* Clear, descriptive title
* Steps to reproduce the problem
* Expected behavior vs. actual behavior
* Environment details (OS, version, browser, etc.)
* Screenshots or error messages if applicable
* Possible solutions if you have ideas

## Suggesting Enhancements and submitting PRs

We welcome feature suggestions! When proposing an enhancement or raising a PR:

* Use a clear, descriptive title
* Provide detailed description of the proposed feature
* Explain the motivation - why is this useful?
* Describe alternatives you've considered
* Include mockups or examples if applicable
* Follow the code standards, style guide and PR template in each repositories `CONTRIBUTING.md`
